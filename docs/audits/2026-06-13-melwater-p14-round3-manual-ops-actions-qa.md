# Melwater P14 Round 3 Manual Ops Actions QA

Date: 2026-06-13

## Scope

P14 round 3 moves the Ops page from read-only visibility to controlled manual operations:

- Trigger an API-level review-state backup from the Ops page
- Generate an application-side Ops report from the Ops page
- Download the latest Markdown Ops report
- Surface certificate expiry days in `/ops` and the UI
- Preserve the security boundary by not mounting Docker socket or controlling other applications

## Implementation

Backend:

- `POST /api/review-state/ops/backup`
  - Requires `admin`.
  - Creates a review-state API backup through `store.createBackup`.
  - Regenerates the latest Ops report after backup.
- `POST /api/review-state/ops/report`
  - Requires `admin`.
  - Generates JSON and Markdown Ops reports from current `/ops` state.
  - Writes dated report files and latest aliases.
- `GET /api/review-state/ops/report/latest.md`
  - Requires `viewer`.
  - Returns the latest Markdown report for download.
- `GET /api/review-state/ops/report/latest.json`
  - Requires `viewer`.
  - Returns the latest JSON report.
- `/api/review-state/ops`
  - Adds certificate summary:
    - `host`
    - `notAfter`
    - `daysRemaining`

Frontend:

- Ops page adds manual action buttons:
  - `触发备份`
  - `生成 report`
  - `下载 report`
- Ops page shows:
  - certificate days remaining
  - certificate `notAfter`
  - manual action result messages

Docker:

- The API container can write only to the Melwater backup mount:

```yaml
${MELWATER_BACKUP_HOST_ROOT:-/opt/melwater-ana/backups}:/ops/backups
```

- Docker socket is not mounted.
- Other application directories are not mounted.

## Remaining Debt

- The manual backup is an API-level review-state backup, not the host tar.gz backup produced by `melwater-backup.sh`.
- A future hardening step could add signed one-shot host jobs instead of granting broader app permissions.
- After Feishu/WeCom credentials are ready, manual report generation can also trigger notification delivery.
